# Resource Map

Use these references carefully. Do not copy code verbatim. Implement minimal examples before copying any large structures.

## 1. Documentation & Structure
- **[cozyGarage/philosophers](https://github.com/cozyGarage/philosophers)**: Our benchmark for documentation shape, learning flow, and the module/reference split.

## 2. General Cub3D Repositories
- **[derfleck/cub3d](https://github.com/derfleck/cub3d)**: A resource shelf for Lode, Permadi, flood fill, Black Book, and MLX links.
- **[Ry4nnnn/42_Cub3d](https://github.com/Ry4nnnn/42_Cub3d)**: Good for comparing explanation structures. This README correctly separates project introduction from algorithm notes and includes a note on the fisheye fix.
- **[melmut-42/cub3D](https://github.com/melmut-42/cub3D)**: Late-stage reference (bonus track) for robust parser, minimap, doors, mouse input, and cleanup ideas.

## 3. Mathematical Theory (Raycasting)
- **[Lode's Computer Graphics Tutorial](https://lodev.org/cgtutor/raycasting.html)**: The primary math reference for DDA, delta distances, and perpendicular distance calculation.
- **[Permadi Ray-Casting Tutorial](https://permadi.com/1996/05/ray-casting-tutorial-table-of-contents/)**: Excellent for visual intuition. *Note: Do not redistribute their images. Link them or recreate your own.*

## 4. MiniLibX (MLX)
- **[42 Docs MiniLibX Guide](https://aurelienbrabant.fr/blog/getting-started-with-the-minilibx)**: MLX basics, image buffer, pixel access, and setup. Explains why buffered drawing is preferred over pixel-by-pixel rendering.

## 5. Historical Context
- **Fabien Sanglard's Game Engine Documentation**: Good historical context explaining why early raycasters often used fixed-point math due to the high cost of floating-point operations at the time, although we aren't restricted to it today.
